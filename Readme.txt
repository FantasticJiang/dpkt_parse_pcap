dpkt是通过逐层拆包的逻辑解析数据包，必须先获取链路层，再获取网络层、传输层、应用层。所以对于pcapdroid抓取的，没有二层报头的文件，dpkt无法直接解析，要先用tcprewrite添加二层报头。优点是能直接解析非知名端口的HTTP/TLS，并且据评估报文读取速度远快于scapy。

scapy读取一个packet后，可以直接获取该报文具备的任意一层，无需逐层拆包，可以直接解析缺失二层报头的文件。scapy的不足可能在于，对HTTP、TLS等知名协议的识别解析依赖于端口，无法识别非知名端口的HTTP/TLS流量，解析时会当做普通TCP流量处理。（暂未查到scapy解析非知名端口HTTP/TLS的方式）
